[EXPERIMENTAL] Support for HTTPREQ based record updates.

Based on: https://github.com/go-acme/lego/tree/master/providers/dns/httpreq

This API is available under https://api.mydnshost.co.uk/experimental/httpreq/.

This also works for acme.sh using the acmeproxy method.

Methods Available

  • GET /ping
  • POST /present
  • POST /cleanup 

/ping
method

The 
/ping
endpoint behaves the same as the version 1.0 API 
/ping
method. 

/present
and 
/cleanup
methods

Both the 
/present
and 
/cleanup
endpoints take JSON formatted data in the format: 
{
	"fqdn": "_acme-challenge.example1.org",
	"value": "LHDhK3oGRvkiefQnx7OOczTY5Tic_xZ6HcMOc_gmtoM"
}

Calling either endpoint will delete all TXT records with the given record name. The 
/present
endpoint will then create a new TXT record with value given (or do nothing if no value is given.)

The response provided is similar to that from providing records to delete and create via the version 1.0 API 
/domain/example1.org/records
method.

Authentication

Authentication is exclusively performed via basic auth.

The username provided can be either a domain name or email address.

If the username is an email address, then the password must be a valid API KEY for that user.

If the username is a domain name, then the password must be a valid DOMAIN KEY for that domain.
